Информация о проекте

CRM, CRM-система, сокращение от англ. Customer Relationship Management - система предназначенная для управления взаимоотношениями с клиентами.

R-CRM представляет из себя прикладное программное обеспечение для организаций, предназначенное для автоматизации стратегий взаимодействия с заказчиками (клиентами), в частности, для повышения уровня продаж, оптимизации маркетинга и улучшения обслуживания клиентов путём сохранения информации о клиентах и истории взаимоотношений с ними, установления и улучшения бизнес-процессов и последующего анализа результатов.

R-CRM предполагается как система предназначенная для управления клиентским обслуживанием и колл-центрами (система по обработке обращений абонентов, фиксации и дальнейшей работе с обращениями клиентов). Классификация по уровню обработки информации: операционный CRM (регистрация и оперативный доступ к первичной информации по событиям, компаниям, проектам, контактам); аналитический CRM (отчётность и анализ информации в различных разрезах (воронка продаж, анализ результатов маркетинговых мероприятий, анализ эффективности продаж в разрезе продуктов, сегментов клиентов, регионов и другие возможные варианты)).

R-CRM имеет следующие разделы

  • Задания (тикеты)
  • Пользователи (раздел управления пользователями)
  • Клиенты (раздел управления клиентами)
  • Статистика (раздел статистики)

    • Проекты, используемые в R-CRM

    1. Python (High-level programming language)
    2. Django (Web Framework)
    3. PostgreSQL (DataBase)
    4. SQlite3 (DataBase)
    5. Foundation (CSS, JS Framework)
    6. JQuery and JQuery-UI (CSS, JS Framework)

    Зависимости

    R-CRM использует интерпритатор python версии 3 и Django версии 1.8.4.

    Инструкция по развёртыванию

    1. Необходимо выбрать и, при необходимости, подготовить СУБД. Настройки СУБД находятся в файле rcrm_django/settings.py, в разделе "DATABASES". По умолчанию выбрана SQLite СУБД. После выбора СУБД, создаём структуру СУБД командой python3 manage.py migrate.
      *Примечание: выполнять команды нужно в корневой директории проекта, там где находиться файл manage.py!
    2. Далее необходимо заполнить созданную базу, необходимыми для нормальной работы программы, данными - справочниками, информацией о статусах и т.д. python3 manage.py loaddata crm/fixtures/initial-priority.json.
    3. Создаём cуперпользователя python3 manage.py createsuperuser, скрипт запросит логин и пароль cуперпользователя, так же можно задать адрес электронной почты. Учётная запись суперпользователя - учётная запись с максимально возможными привилегиями в системе, держите пароль от неё в тайне.
    4. Для тестового использования Вы можете воспользоваться интегрированным с Django веб-сервером для разработки - просто выполните команду python3 manage.py runserver 8000 (8000 - номер порта, на котором будет запущен сервер) и откройте браузер по адресу http://localhost:8000.
      *Примечание: не используйте данный способ на боевом сервере! Для боевого сервера можете использовать связку nginx + uwsgi, например.
    5. RCRM готова к работе!
